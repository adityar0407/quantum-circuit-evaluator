# Repository Cleanup Inventory

This repo now has three intended layers:

1. Scientific core under `backend/target_creation`, `backend/hardware`,
   `backend/IR`, `backend/metrics`, and `backend/pass_managers`.
2. Local app/API layer under `backend/api` and `backend/services`.
3. Frontend app under `frontend`.

## Keep

- `backend/target_creation/target.py`: canonical `FTarget` implementation.
- `backend/FaultTolerantTarget.py`: small user-facing wrapper.
- `backend/hardware/connectivity.py`: coupling-map generation and IBM CSV loading.
- `backend/IR/*`: circuit/QASM loading and metadata helpers.
- `backend/metrics/*`: benchmark metric helpers.
- `backend/pass_managers/*`: modular pass manager components for future custom pipelines.
- `backend/connectivity_maps/*.csv`: source data for IBM Fez/Torino maps.
- `examples/01_Target_Creation.ipynb`: current demonstration artifact.
- `Project Report .pdf`: preserved report artifact.

## Obsolete Or Stale

- `examples/iterate_network_configurations.py`: marked `old code`, uses stale imports,
  and uses the old list-based profile schema.
- `backend/main_pipeline.py`: exploratory script with stale top-level imports. The new
  app path should use `backend/services/transpilation_service.py`.
- `backend/tests/test_all_targets.py`: was stale; it has been updated to the current
  `backend.*` import path and current profile schema.
- `docs/scientific_benchmarking_refactor_plan.md`: useful historically, but it still
  references a `src/` layout that no longer exists.
- `docs/max_next_steps.md`: stale pytest path and informal notes.
- `examples/jsons/blueprint.json`: stale placeholder gate names that are not valid
  Qiskit gate classes.
- `Fault_tolerant_Target.egg-info/`: generated packaging metadata. It should not be
  edited by hand and can be regenerated by packaging commands.

## Recommended Next Cleanup Pass

1. Move obsolete scripts into `archive/` or delete them after confirming they are not
   needed for the report/demo.
2. Replace `examples/jsons/blueprint.json` with a valid starter config.
3. Convert `backend/main_pipeline.py` into either a maintained CLI or remove it.
4. Update docs to reference the `backend/` layout and local app workflow.
5. Add focused tests for `backend/services` and API request validation.
